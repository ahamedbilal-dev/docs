There's no single right way to follow this course — but there are strategies that work much better than others. This page covers how the video and handbook fit together, how to adjust when the pace feels off, how to approach code examples, and how to use AI assistants as a learning accelerator throughout your journey.

## Video and handbook together

This course exists as both a [YouTube video](https://youtu.be/ygXn5nV5qFc) and this written handbook. Each format has strengths:

- **The video** walks you through every concept in real-time, showing exactly what to type, what to click, and how things behave when they run.
- **The handbook** contains everything from the video *plus* additional details, deeper explanations, and code snippets you can copy directly rather than transcribing from the screen.

The most effective approach is to **keep this handbook open while watching**. When Dave references a section, jump to it here. When you see code you want to experiment with, copy it from the handbook rather than pausing and rewinding. Use both together and you'll absorb the material significantly faster than using either alone.

## Adjust to your pace

Everyone learns at a different speed, and the same person can find different topics easy or hard. Here's how to adapt:

<Tabs>
  <Tab title="Going too fast">
    If a concept isn't landing, you have several options:

    - **Pause and read**. Find the corresponding handbook chapter and read through it carefully. The written format often makes things click when the video doesn't.
    - **Use AI as a tutor**. Paste the concept or code into ChatGPT or Claude and ask for a simpler explanation. Something like: *"Explain Python list comprehensions as if I've never programmed before."* AI assistants are available 24/7 and infinitely patient — use them.
    - **Give it another pass**. Some concepts need to be encountered two or three times before they fully sink in. That's completely normal. Move forward and come back — context makes earlier topics clearer in retrospect.
    - **Slow down the video**. YouTube lets you play at 0.75x speed. This sounds obvious but it helps more than people expect.

    <Note>
      Feeling confused doesn't mean you're not cut out for this. It means you're at the edge of your current understanding — exactly where learning happens.
    </Note>
  </Tab>
  <Tab title="Going too slow">
    If the pace feels comfortable and you want to move faster:

    - **Use video timestamps**. The YouTube video has timestamps for every major section. Jump straight to what you need rather than watching from the beginning.
    - **Speed up playback**. Watch at 1.5x or 2x speed — many experienced learners use this as their default.
    - **Skim the handbook**. Read through a section's headings and code blocks first to decide whether you need the full explanation or just a quick refresher.
    - **Skip the basics you know**. If you're coming from another language, the early sections on variables, loops, and conditionals might be review. It's fine to skim or skip them and pick up where things get unfamiliar.

    <Tip>
      Even if you're moving quickly, don't skip the workflow and tooling sections. The VS Code setup, Git workflow, and virtual environment chapters contain patterns that are different from other languages.
    </Tip>
  </Tab>
</Tabs>

## About the code examples

As you work through the course, you'll encounter increasingly complex code — and that's intentional. The goal isn't to understand every single line immediately. It's *exposure to what's possible*.

We start simple and build complexity gradually. Real understanding comes from writing your own code, not just watching. When you see a large example and think "this is too much," that reaction is completely normal. You're building foundations, and the gaps will fill in as you practice and build your own projects.

<Info>
  Copy code from the handbook rather than typing it from the video. This saves time, avoids typos, and lets you focus on understanding what the code does rather than transcribing it accurately.
</Info>

## Use AI to learn faster

AI assistants are incredibly powerful learning tools when used with intention. Here are three example prompts you can use right now:

```text Example prompts
"Explain this Python code in simple terms: [paste code here]"

"What does this error mean and how do I fix it? [paste error message]"

"Show me 3 different ways to solve this problem in Python: [describe the problem]"
```

The key is giving the AI enough context. Don't just say "my code is broken." Share the actual code, the actual error, and what you were trying to do. You'll get dramatically better answers.

See the [AI Assistants](/introduction/ai-assistants) page for a full breakdown of tools, prompt strategies, and a custom Python tutor system prompt you can use with ChatGPT or Claude.

## This handbook stays updated

Unlike the video — which is fixed once published — this handbook is updated regularly. If something in the video no longer works (outdated syntax, a changed tool interface, a deprecated library), check the handbook first. When students surface issues or tools change, the relevant pages get updated so everyone benefits.

## How to approach this course

Keep these principles in mind throughout:

- **Watch actively and code along** — don't just observe
- **Make mistakes and fix them** — errors are your best teachers
- **Use the handbook to go deeper** and copy code when it saves time
- **Ask AI when you're stuck** — it's your 24/7 tutor
- **After each major section, build something** — even a small variation on what you just learned
- **Accept that you don't need to master everything before moving on** — understanding deepens with practice
