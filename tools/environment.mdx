Every AI application you build will need at least one secret — an OpenAI API key, a database password, a third-party token. The most common beginner mistake is pasting those secrets directly into Python code. That makes them visible to anyone who reads your code, anyone who clones your repo, and — if you push to a public GitHub repository — the entire internet. This section shows you the right way to handle secrets from day one.

## Why hardcoding secrets is dangerous

```python
# ❌ NEVER do this
api_key = "sk-1234567890abcdef"      # Anyone reading your code can see this!
db_password = "super_secret_pass"   # This ends up on GitHub if you push
```

The problems compound quickly:

- Secrets are visible to every collaborator
- They're committed to Git history — even after deletion
- Different environments (laptop, server, CI) need different values
- You can't share code publicly without first scrubbing secrets

## The solution: environment variables

Environment variables let you keep secrets in a separate location that's outside your codebase entirely. Your Python code reads the value at runtime — it never stores the secret itself.

```python
import os

# ✅ Read the secret from the environment — not from code
api_key = os.environ.get("OPENAI_API_KEY")
```

## The golden rule

<Warning>
  Add `.env` to your `.gitignore` file **before** your first commit. If a secret
  is ever pushed to GitHub — even briefly — treat it as compromised and regenerate
  it immediately.
</Warning>
