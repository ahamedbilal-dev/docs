Programming is simply the act of writing instructions for a computer to follow. Computers are powerful machines, but they aren't smart — they need detailed, step-by-step directions for everything they do. Programming languages like Python give us a way to write those directions in a form that both humans can read and computers can execute.

## What Is Programming?

Think of programming like writing a recipe. A recipe tells a cook exactly what to do, in what order, and with what ingredients. Code does the same for a computer — it leaves nothing to guesswork. Computers only understand binary (1s and 0s) at the lowest level, so programming languages act as a readable middle ground.

Here's how that translation chain works:

1. **You write** code in Python (looks like English)
2. **Python translates** your code into machine instructions
3. **Computer executes** those instructions

## Your First Taste of Python

One of the best ways to understand Python is to just look at it. Here's a short program that demonstrates variables, string formatting, and a simple decision:

```python
# This is Python code — it almost reads like English!
name = "Sarah"
age = 25
print(f"Hello, my name is {name} and I am {age} years old")

# Making a simple decision
if age >= 18:
    print("I can vote!")
else:
    print("I'm too young to vote")
```

Notice how readable that is? No curly braces, no semicolons, no noise. That readability is Python's superpower, and it's why the language is so popular as a first language and in the AI space.

## Python Versions Explained

There are two major version lines of Python, but the choice is simple:

<Tabs>
  <Tab title="Python 3 (Use This)">
    Python 3 is the current, actively maintained version of the language. All new projects, libraries, and tutorials use Python 3. This is what you'll learn and use throughout this course. You can see the current supported versions at [devguide.python.org/versions](https://devguide.python.org/versions/).
  </Tab>
  <Tab title="Python 2 (Avoid)">
    Python 2 reached end-of-life in January 2020 and is no longer supported. You may encounter old code or tutorials that mention Python 2 — ignore them and find newer resources. Using Python 2 today is like running Windows XP: technically possible, but a bad idea.
  </Tab>
</Tabs>

<Warning>
  Always use **Python 3**. If you see a tutorial or code snippet specifically referencing Python 2 syntax (such as `print "hello"` without parentheses), find a more up-to-date resource.
</Warning>
