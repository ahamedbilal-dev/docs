Real Python projects quickly grow beyond a single file. As soon as you're working with data, generating output, or splitting logic across modules, you need a clear folder structure — one that makes it obvious where things live and keeps your code easy to maintain. This page walks you through setting up a clean, practical project layout from scratch.

## Working with Multiple Files

So far you may have written single Python files, but real projects have multiple files, folders, and resources. Good organization makes your code easier to understand, navigate, and share with others. Let's set up a proper workspace for a sales analysis project.

## Recommended Folder Structure

Here is the recommended layout for a self-contained Python project like a sales analysis:

```
python-for-ai/
├── hello.py                  # Your existing practice file
└── sales-analysis/           # New project folder
    ├── analyzer.py           # Your main script
    ├── helpers.py            # Reusable helper functions
    ├── data/                 # Input files (CSV, JSON, etc.)
    │   └── sales.csv
    └── output/               # Generated files and results
```

<Note>
  Create folders in VS Code by right-clicking in the Explorer panel and selecting **New Folder**, or click the new-folder icon at the top of the Explorer panel.
</Note>

The key principle here is that your **main script sits at the project root**, alongside any helper modules, while `data/` and `output/` serve as clearly labelled compartments for input and output files respectively.

## Understanding File Paths

When your script opens a file, Python looks for it relative to your **current working directory** — the folder Python is running from. With the structure above, your script `analyzer.py` lives in `sales-analysis/`, so all paths start from there:

```python
# From analyzer.py inside sales-analysis/

"data/sales.csv"        # ✅ Look in the data subfolder
"output/report.json"    # ✅ Save to the output subfolder

# Visualising where things are:
# sales-analysis/
#   ├── analyzer.py        <- Your script is here
#   ├── data/sales.csv     <- Your data is here
#   └── output/            <- Results go here
```

<Tip>
  Always keep your main script at the same level as `data/` and `output/`. This makes paths stay short and consistent regardless of how you run the script.
</Tip>

## Creating Your First Script

In the `sales-analysis/` folder (not inside a subfolder), create `analyzer.py`. This starter script checks that Python is running from the right place and that your data file exists before doing any real work:

```python
import os

# Check if we're in the right place
print("Current directory:", os.getcwd())

# Check if our data file exists
data_path = "data/sales.csv"

if os.path.exists(data_path):
    print(f"✅ Found {data_path}")
else:
    print(f"❌ Cannot find {data_path}")
    print("Make sure you're running from the sales-analysis folder!")
```

Run this first. If you see `❌ Cannot find data/sales.csv`, you're either missing the file or running the script from the wrong directory.

## Best Practices

Following a few simple conventions will save you headaches as your projects grow:

<Accordion title="Keep data separate from code">
  Never mix `.csv`, `.json`, or other data files in the same folder as your Python scripts. Put them in a dedicated `data/` folder. This makes it obvious what is source code and what is input data, and prevents you from accidentally committing large data files to version control.
</Accordion>

<Accordion title="Use clear, predictable folder names">
  Stick to `data/` for input files and `output/` (or `results/`) for generated files. Anyone reading your project — including future you — will immediately know where to look.
</Accordion>

<Accordion title="Keep the main script at project level">
  Place your primary script (`analyzer.py`, `main.py`, etc.) directly inside the project folder, not inside a subfolder. This keeps file paths short and ensures both the play button and Shift+Enter work without extra configuration.
</Accordion>

<Accordion title="Test as you go">
  Use Shift+Enter to run your script incrementally. Verifying each step before moving to the next catches path and logic errors early, before they compound into harder-to-debug problems.
</Accordion>

<Accordion title="Check your working directory first">
  If something isn't working, add `print(os.getcwd())` at the top of your script. Nine times out of ten, a `FileNotFoundError` is caused by running the script from an unexpected directory.
</Accordion>
