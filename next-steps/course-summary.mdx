## Skills that transfer directly to AI work

Everything you learned was chosen because it matters for AI development specifically:

<Steps>
  <Step title="Reading and writing files">
    AI workflows constantly read prompts, write outputs, parse CSVs, and log
    results. You can do all of that.
  </Step>
  <Step title="Calling APIs">
    OpenAI, Anthropic, Google, Hugging Face — all AI services are accessed via
    HTTP APIs. You know how to call them, handle errors, and process responses.
  </Step>
  <Step title="Managing secrets">
    API keys are everywhere in AI development. You know the right way to handle
    them with `.env` files so they never end up in your code.
  </Step>
  <Step title="Using Git and GitHub">
    AI projects are shared on GitHub. You can clone repos, contribute changes,
    and back up your own work professionally.
  </Step>
  <Step title="Managing dependencies with uv">
    AI libraries like LangChain, OpenAI SDK, and others come with complex
    dependency trees. uv handles them reliably and fast.
  </Step>
</Steps>

## From beginner to builder

The shift that matters most isn't the specific commands you learned — it's that you now have the foundation to *figure things out*. You know how to read error messages, search for answers, read documentation, and iterate. Those meta-skills are what separate developers who grow quickly from those who stay stuck.

<Note>
  The best way to keep learning is to build something real. Pick a project that
  excites you — a tool that automates something tedious, a small AI app, a
  script that solves a problem you actually have — and build it. You'll learn
  more from one real project than from ten tutorials.
</Note>
